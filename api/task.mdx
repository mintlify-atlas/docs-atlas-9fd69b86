---
title: 'Tasks'
description: 'Endpoints for managing background tasks, schedules, and notifications.'
---

Tube Archivist runs all long-running operations as background Celery tasks. Use these endpoints to trigger tasks, check their status, configure schedules, and set up webhook notifications. All task endpoints require admin privileges.

## Available task names

The following task names are valid for use in task and schedule endpoints:

| Task name | Description | Can start via API | Can stop via API |
|---|---|---|---|
| `update_subscribed` | Rescan subscriptions | Yes | Yes |
| `download_pending` | Start downloading queued videos | Yes | Yes |
| `extract_download` | Add to download queue | No | Yes |
| `check_reindex` | Reindex documents | No | No |
| `manual_import` | Import local video files | No | No |
| `run_backup` | Create an index backup | Yes | No |
| `restore_backup` | Restore from backup | No | No |
| `rescan_filesystem` | Rescan the filesystem | No | No |
| `thumbnail_check` | Check and repair thumbnails | Yes | No |
| `resync_metadata` | Sync metadata to media files | Yes | No |
| `index_playlists` | Index channel playlists | No | No |
| `subscribe_to` | Add subscriptions | No | No |
| `version_check` | Check for new Tube Archivist version | No | No |

## List all task results

`GET /api/task/by-name/`

Returns the most recent stored results for all known tasks.

<CodeGroup>

```bash curl
curl http://your-host:8000/api/task/by-name/ \
  -H "Authorization: Token <your-token>"
```

```python Python
import requests

response = requests.get(
    "http://your-host:8000/api/task/by-name/",
    headers={"Authorization": "Token <your-token>"},
)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  "http://your-host:8000/api/task/by-name/",
  { headers: { "Authorization": "Token <your-token>" } }
);
const data = await response.json();
```

</CodeGroup>

Returns an array of task result objects:

```json
[
  {
    "task_id": "c9a1b2d3-e4f5-6789-abcd-ef0123456789",
    "name": "download_pending",
    "status": "SUCCESS",
    "result": "null",
    "traceback": null,
    "date_done": "2024-01-16T10:30:00",
    "args": [],
    "kwargs": {},
    "worker": "celery@worker",
    "retries": 0,
    "queue": "celery"
  }
]
```

## List results for a named task

`GET /api/task/by-name/<task_name>/`

Returns all stored results for a specific task.

<CodeGroup>

```bash curl
curl http://your-host:8000/api/task/by-name/download_pending/ \
  -H "Authorization: Token <your-token>"
```

```python Python
import requests

response = requests.get(
    "http://your-host:8000/api/task/by-name/download_pending/",
    headers={"Authorization": "Token <your-token>"},
)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  "http://your-host:8000/api/task/by-name/download_pending/",
  { headers: { "Authorization": "Token <your-token>" } }
);
const data = await response.json();
```

</CodeGroup>

Returns `200 OK` with an array of task result objects, or `404` if the task name is not recognized.

### Task result fields

<ResponseField name="task_id" type="string">
  Unique task identifier (UUID).
</ResponseField>

<ResponseField name="name" type="string">
  Task name (e.g., `download_pending`).
</ResponseField>

<ResponseField name="status" type="string">
  One of `PENDING`, `STARTED`, `SUCCESS`, `FAILURE`, `RETRY`, or `REVOKED`.
</ResponseField>

<ResponseField name="result" type="string">
  Serialized task return value, or `null`.
</ResponseField>

<ResponseField name="traceback" type="string">
  Error traceback if the task failed, otherwise `null`.
</ResponseField>

<ResponseField name="date_done" type="string">
  ISO 8601 timestamp of completion.
</ResponseField>

<ResponseField name="args" type="array">
  Positional arguments the task was called with.
</ResponseField>

<ResponseField name="kwargs" type="object">
  Keyword arguments the task was called with.
</ResponseField>

<ResponseField name="worker" type="string">
  Celery worker hostname.
</ResponseField>

<ResponseField name="retries" type="integer">
  Number of times the task was retried.
</ResponseField>

<ResponseField name="queue" type="string">
  Queue name the task ran in.
</ResponseField>

## Trigger a named task

`POST /api/task/by-name/<task_name>/`

Starts a new background task. Only tasks with `api_start: true` can be triggered this way (see the table above).

<CodeGroup>

```bash curl
curl -X POST http://your-host:8000/api/task/by-name/update_subscribed/ \
  -H "Authorization: Token <your-token>"
```

```python Python
import requests

response = requests.post(
    "http://your-host:8000/api/task/by-name/update_subscribed/",
    headers={"Authorization": "Token <your-token>"},
)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  "http://your-host:8000/api/task/by-name/update_subscribed/",
  {
    method: "POST",
    headers: { "Authorization": "Token <your-token>" },
  }
);
const data = await response.json();
```

</CodeGroup>

**Response** `200 OK`

```json
{
  "message": "task started",
  "task_id": "c9a1b2d3-e4f5-6789-abcd-ef0123456789"
}
```

Returns `404` if the task name is unknown or cannot be started via the API.

## Send a command to a task

`POST /api/task/by-id/<task_id>/`

Stops or kills a running task. Only tasks with `api_stop: true` support this (see the table above).

<ParamField body="command" type="string" required>
  Command to send. One of:
  - `stop` — gracefully stop the task
  - `kill` — immediately terminate the task
</ParamField>

<CodeGroup>

```bash curl
curl -X POST http://your-host:8000/api/task/by-id/c9a1b2d3-.../ \
  -H "Authorization: Token <your-token>" \
  -H "Content-Type: application/json" \
  -d '{"command": "stop"}'
```

```python Python
import requests

task_id = "c9a1b2d3-e4f5-6789-abcd-ef0123456789"
response = requests.post(
    f"http://your-host:8000/api/task/by-id/{task_id}/",
    headers={"Authorization": "Token <your-token>"},
    json={"command": "stop"},
)
print(response.status_code)  # 204
```

```javascript JavaScript
const response = await fetch(
  `http://your-host:8000/api/task/by-id/${taskId}/`,
  {
    method: "POST",
    headers: {
      "Authorization": "Token <your-token>",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ command: "stop" }),
  }
);
```

</CodeGroup>

Returns `204 No Content`, `400` if the task cannot be stopped, or `404` if the task is not found.

</ResponseField>

## Create or update a schedule

`POST /api/task/schedule/<task_name>/`

Creates or updates the cron schedule for a task.

<ParamField body="schedule" type="string">
  Cron expression with 5 fields: `minute hour day month weekday` (e.g., `"0 3 * * *"` for 3 AM daily).
</ParamField>

<ParamField body="config" type="object">
  Optional task configuration. For example, `{"days": 90}` to configure retention for cleanup tasks.
</ParamField>

<Note>
  At least one of `schedule` or `config` must be provided.
</Note>

<CodeGroup>

```bash curl
curl -X POST http://your-host:8000/api/task/schedule/update_subscribed/ \
  -H "Authorization: Token <your-token>" \
  -H "Content-Type: application/json" \
  -d '{"schedule": "0 */6 * * *"}'
```

```python Python
import requests

response = requests.post(
    "http://your-host:8000/api/task/schedule/update_subscribed/",
    headers={"Authorization": "Token <your-token>"},
    json={"schedule": "0 */6 * * *"},
)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  "http://your-host:8000/api/task/schedule/update_subscribed/",
  {
    method: "POST",
    headers: {
      "Authorization": "Token <your-token>",
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ schedule: "0 */6 * * *" }),
  }
);
const data = await response.json();
```

</CodeGroup>

Returns `200 OK` with the created or updated schedule object.

## Add a notification URL

`POST /api/task/notification/`

Adds a webhook URL to a task's notification list.

<ParamField body="task_name" type="string" required>
  Task name to attach the notification to (see the task name table above).
</ParamField>

<ParamField body="url" type="string" required>
  Webhook URL to notify when the task completes.
</ParamField>

<CodeGroup>

```bash curl
curl -X POST http://your-host:8000/api/task/notification/ \
  -H "Authorization: Token <your-token>" \
  -H "Content-Type: application/json" \
  -d '{"task_name": "download_pending", "url": "https://example.com/webhook"}'
```

```python Python
import requests

response = requests.post(
    "http://your-host:8000/api/task/notification/",
    headers={"Authorization": "Token <your-token>"},
    json={"task_name": "download_pending", "url": "https://example.com/webhook"},
)
print(response.json())
```

```javascript JavaScript
const response = await fetch("http://your-host:8000/api/task/notification/", {
  method: "POST",
  headers: {
    "Authorization": "Token <your-token>",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    task_name: "download_pending",
    url: "https://example.com/webhook",
  }),
});
const data = await response.json();
```

</CodeGroup>

Returns `200 OK` with the full updated notification configuration.
